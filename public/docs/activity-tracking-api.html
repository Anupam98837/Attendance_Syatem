Activity Tracking API

Complete reference for the employee activity tracking endpoints — GPS traversal, WiFi events, office presence, movement detection, and app state — with app integration guides for Android, iOS, and Browser.

✓ REST JSON API Bearer Token Auth Background Tracking Offline-first

🔐Authentication

All endpoints require a Bearer token in the Authorization header.

# Every request must include:
Authorization: Bearer <your_jwt_token>
Accept: application/json
ℹ️ The token is obtained from POST /api/auth/login and stored in the app's secure storage. Employee tokens have access to the /mobile endpoints. HR/Admin tokens access the /hr endpoints.

🌐Base URL

/api/attendance

# Employee endpoints  →  /api/attendance/mobile/activity-log
# HR/Admin endpoints  →  /api/attendance/hr/activity-logs

📤POST — Submit Activity Log

Submit or update the daily activity log for the authenticated employee. Safe to call multiple times — it upserts on (user_id, attendance_date), so the latest payload always wins.

POST /api/attendance/mobile/activity-log role: employee

Upserts the activity log for the given date. Send this after checkout, periodically during the day, or when the app comes back online after offline usage.

Returns the full activity log including GPS path array, WiFi networks, raw event stream, and linked attendance record data.

📍GPS Path Format

The gps_path field is an array of waypoint objects. Collect a new waypoint whenever the device moves more than 5–10 meters or every 30–60 seconds while moving.

Waypoint Object

FieldTypeDescription
latfloatLatitude in decimal degrees (e.g. 12.97161).
lngfloatLongitude in decimal degrees (e.g. 77.59461).
accintegerHorizontal accuracy in meters (±).
tsstring (ISO 8601)UTC timestamp of the GPS fix.
spd_kmhfloatSpeed at this point in km/h. Use 0 or omit if stationary.
altfloatAltitude in meters (optional).
is_movingbooleantrue if the device was moving at this point.

Example

"gps_path": [
  {
    "lat": 12.97161, "lng": 77.59461,
    "acc": 5, "ts": "2026-05-09T09:02:15Z",
    "spd_kmh": 0.0, "alt": 920.5, "is_moving": false
  },
  {
    "lat": 12.97249, "lng": 77.59512,
    "acc": 8, "ts": "2026-05-09T09:15:30Z",
    "spd_kmh": 4.2, "alt": 921.0, "is_moving": true
  }
]
⚠️ Limit path size: Cap at 500 waypoints per day before syncing. Use distance-based sampling (only add a point if the device moved >10m) to keep the array small without losing path fidelity.

📝Raw Event Stream

An ordered log of all significant tracking events. Stored for debugging — HR will never see this unless they access the raw API. Keep it trimmed to the last 500 events.

tsstring (ISO 8601)Event timestamp in UTC.
msgstringHuman-readable event description.
payloadobjectOptional extra data for this event.

  { "type": "wifi", "ts": "2026-05-09T09:00:05Z", "msg": "WiFi connected",
    "payload": { "ssid": "Office-Corp" } },
  { "type": "office", "ts": "2026-05-09T09:01:00Z", "msg": "Entered office geofence" },
  { "type": "move", "ts": "2026-05-09T11:30:00Z", "msg": "Moved 42m",

  { "type": "battery", "ts": "2026-05-09T17:00:00Z", "msg": "Battery low",
    "payload": { "level": 18 } }

📱Background Tracking Guide

The app must track GPS, WiFi, and office presence even when the user locks their screen or switches to another app. Here's how to implement it correctly on each platform.

Core Tracking Responsibilities

Start tracking on check-in punch

When the user checks in, start the GPS watcher, WiFi listener, and geofence monitor. Record session_start and battery_start_percent. Register a background service / WorkManager job.

Collect events throughout the day

Store all events in a local SQLite database (not memory). Include GPS waypoints on every significant movement (>10m or every 60s while moving). Log WiFi SSID changes, geofence transitions, and battery checkpoints.

Compute summary counters locally

Maintain running counters for all integer fields (gps_connect_count, wifi_switch_count, etc.) in a single local record. Update them in real-time as events arrive.

Sync on checkout OR every 15 minutes

After checkout, assemble the full payload and POST /api/attendance/mobile/activity-log. Also schedule a periodic 15-minute sync during the day so data isn't lost if the app crashes.

Stop tracking on checkout

Remove the GPS watcher, unregister WiFi listeners, stop the background service. Record session_end and battery_end_percent. Do a final sync.

🗄️Local Database Schema

Store data locally in SQLite before syncing. One row per work day per device.

-- activity_log (one row per day)
CREATE TABLE activity_log (
  id                            INTEGER PRIMARY KEY,
  attendance_date               TEXT NOT NULL,          -- YYYY-MM-DD
  session_start                 TEXT,                   -- ISO 8601
  session_end                   TEXT,
  gps_connect_count             INTEGER DEFAULT 0,
  gps_disconnect_count          INTEGER DEFAULT 0,
  total_distance_meters         INTEGER DEFAULT 0,
  is_traveling                  INTEGER DEFAULT 0,      -- 0/1
  time_traveling_minutes        INTEGER DEFAULT 0,
  time_stationary_minutes       INTEGER DEFAULT 0,
  max_speed_kmh                 REAL,
  furthest_distance_from_office INTEGER,
  wifi_connect_count            INTEGER DEFAULT 0,
  wifi_disconnect_count         INTEGER DEFAULT 0,
  wifi_switch_count             INTEGER DEFAULT 0,
  office_entry_count            INTEGER DEFAULT 0,
  office_exit_count             INTEGER DEFAULT 0,
  time_inside_office_minutes    INTEGER DEFAULT 0,
  time_outside_office_minutes   INTEGER DEFAULT 0,
  movement_event_count          INTEGER DEFAULT 0,
  idle_streak_max_minutes       INTEGER DEFAULT 0,
  app_foreground_count          INTEGER DEFAULT 0,
  app_background_count          INTEGER DEFAULT 0,
  app_foreground_minutes        INTEGER DEFAULT 0,
  app_background_minutes        INTEGER DEFAULT 0,
  battery_start_percent         INTEGER,
  battery_end_percent           INTEGER,
  synced_at                     TEXT,                   -- null if not yet synced
  created_at                    TEXT DEFAULT (datetime('now'))
);

-- gps_waypoints (separate table, joined at sync time)
CREATE TABLE gps_waypoints (
  id        INTEGER PRIMARY KEY,
  log_date  TEXT NOT NULL,
  lat       REAL, lng REAL, acc INTEGER,
  ts        TEXT, spd_kmh REAL, alt REAL,
  is_moving INTEGER DEFAULT 0
);

-- wifi_events
CREATE TABLE wifi_events (
  id               INTEGER PRIMARY KEY,
  log_date         TEXT NOT NULL,
  ssid             TEXT, bssid TEXT,
  connected_at     TEXT, disconnected_at TEXT,
  duration_seconds INTEGER
);

-- raw_events (append-only, trimmed to last 500)
CREATE TABLE raw_events (
  id       INTEGER PRIMARY KEY,
  log_date TEXT NOT NULL,
  type     TEXT, ts TEXT, msg TEXT, payload TEXT  -- payload as JSON string
);

🔄Sync Flow

// Pseudocode — works the same on Android, iOS, and Browser

function buildSyncPayload(date) {
  const log       = db.query("SELECT * FROM activity_log WHERE attendance_date = ?", date);
  const waypoints = db.query("SELECT * FROM gps_waypoints WHERE log_date = ?", date);
  const wifiNets  = db.query("SELECT * FROM wifi_events WHERE log_date = ?", date);
  const events    = db.query("SELECT * FROM raw_events WHERE log_date = ? ORDER BY ts DESC LIMIT 500", date);

  return {
    ...log,
    gps_path:           waypoints,
    wifi_networks_seen: wifiNets,
    raw_events:         events,
    client_created_at:  log.created_at,
  };
}

async function syncToServer(date) {
  if (!isOnline()) return;          // retry when online
  const payload = buildSyncPayload(date);
  const res = await fetch("/api/attendance/mobile/activity-log", {
    method: "POST",
    headers: { "Authorization": "Bearer " + token, "Content-Type": "application/json" },
    body: JSON.stringify(payload)
  });
  if (res.ok) db.run("UPDATE activity_log SET synced_at = ? WHERE attendance_date = ?", now(), date);
}

// Call this on: checkout, app resume, every 15 minutes, network reconnect

Key Components

// 1. Foreground Service — keeps tracking alive even when screen is off
class ActivityTrackingService : ForegroundService() {
  override fun onStartCommand(...): Int {
    startForeground(NOTIF_ID, buildNotification("Tracking active…"))
    startGPS(); startWiFiMonitor(); startGeofence()
    return START_STICKY     // restart if killed
  }
}

// 2. GPS — request updates via FusedLocationProviderClient
val req = LocationRequest.Builder(Priority.PRIORITY_HIGH_ACCURACY, 30_000L)
  .setMinUpdateDistanceMeters(10f)      // only on 10m+ movement
  .setMaxUpdateDelayMillis(60_000L)    // force at least every 60s while moving
  .build()

// 3. WiFi — register BroadcastReceiver for NETWORK_STATE_CHANGED_ACTION
val filter = IntentFilter(WifiManager.NETWORK_STATE_CHANGED_ACTION)
registerReceiver(wifiReceiver, filter)

// 4. WorkManager — periodic sync every 15 minutes
val work = PeriodicWorkRequestBuilder<SyncWorker>(15, TimeUnit.MINUTES).build()
WorkManager.getInstance(context).enqueueUniquePeriodicWork("activity-sync", KEEP_ALIVE, work)

// 5. Geofence — monitor office location
val geofence = Geofence.Builder()
  .setRequestId("office")
  .setCircularRegion(officeLat, officeLng, 100f)   // 100m radius
  .setTransitionTypes(GEOFENCE_TRANSITION_ENTER or GEOFENCE_TRANSITION_EXIT)
  .setExpirationDuration(NEVER_EXPIRE)
  .build()

// Location Manager setup
let manager = CLLocationManager()
manager.requestAlwaysAuthorization()
manager.allowsBackgroundLocationUpdates = true
manager.pausesLocationUpdatesAutomatically = false
manager.desiredAccuracy = kCLLocationAccuracyBest
manager.distanceFilter = 10         // 10m minimum displacement
manager.startUpdatingLocation()

// Also register for significant-change updates (works when app is killed)
manager.startMonitoringSignificantLocationChanges()

// Geofence — office region
let region = CLCircularRegion(
  center: CLLocationCoordinate2D(latitude: officeLat, longitude: officeLng),
  radius: 100, identifier: "office"
)
region.notifyOnEntry = true; region.notifyOnExit = true
manager.startMonitoring(for: region)

// BGTaskScheduler — periodic sync
BGTaskScheduler.shared.register(forTaskWithIdentifier: "com.app.activity-sync", ...) { task in
  syncToServer(date: today)
  scheduleNextSync()
}

// WiFi — use CNCopyCurrentNetworkInfo and observe changes via reachability
NotificationCenter.default.addObserver(self, selector: #selector(wifiChanged),
  name: .reachabilityChanged, object: nil)

🌐Browser / Web App Integration

⚠️ Browsers cannot detect WiFi SSIDs or network switches. wifi_connect_count and wifi_networks_seen should be left as 0/null when tracking from a web app. Focus on GPS, movement, and visibility state. 
// The employee dashboard auto-starts tracking for today's date.
// Here is the complete browser tracker snippet you can embed in any page:

const TRACKER = {
  session_start: new Date().toISOString(),
  platform: 'web',
  gps_connect_count: 0, gps_disconnect_count: 0,
  gps_path: [], total_distance_meters: 0,
  time_traveling_minutes: 0, time_stationary_minutes: 0,
  movement_event_count: 0, idle_streak_max_minutes: 0,
  app_foreground_count: 0, app_background_count: 0,
  app_foreground_minutes: 0, app_background_minutes: 0,
  raw_events: [],
  _lastPos: null, _lastActivity: Date.now(),
  _watchId: null, _fgStart: Date.now(), _bgStart: null,
};

// Haversine
function haversine(lat1,lng1,lat2,lng2) {
  const R=6371000, dLat=(lat2-lat1)*Math.PI/180, dLng=(lng2-lng1)*Math.PI/180;
  const a=Math.sin(dLat/2)**2+Math.cos(lat1*Math.PI/180)*Math.cos(lat2*Math.PI/180)*Math.sin(dLng/2)**2;
  return R*2*Math.atan2(Math.sqrt(a),Math.sqrt(1-a));
}

// Start GPS watch
TRACKER._watchId = navigator.geolocation.watchPosition(
  pos => {
    const {latitude:lat, longitude:lng, accuracy:acc, speed} = pos.coords;
    const dist = TRACKER._lastPos ? haversine(TRACKER._lastPos.lat, TRACKER._lastPos.lng, lat, lng) : 0;
    if (dist > 5) { TRACKER.total_distance_meters += dist; TRACKER.movement_event_count++; }
    TRACKER.gps_path.push({ lat, lng, acc: Math.round(acc), ts: new Date().toISOString(),
      spd_kmh: speed ? +(speed*3.6).toFixed(1) : 0, is_moving: dist > 5 });
    if (TRACKER.gps_path.length > 200) TRACKER.gps_path.shift();
    TRACKER._lastPos = {lat, lng};
  },
  err => { TRACKER.gps_disconnect_count++; },
  { enableHighAccuracy: true, timeout: 15000, maximumAge: 10000 }
);

// Visibility change (foreground/background proxy)
document.addEventListener('visibilitychange', () => {
  const now = Date.now();
  if (document.hidden) {
    TRACKER.app_background_count++;
    TRACKER.app_foreground_minutes += Math.round((now - TRACKER._fgStart) / 60000);
    TRACKER._bgStart = now;
  } else {
    TRACKER.app_foreground_count++;
    if (TRACKER._bgStart) TRACKER.app_background_minutes += Math.round((now - TRACKER._bgStart) / 60000);
    TRACKER._fgStart = now;
  }
});

// Sync every 5 minutes + on page unload
async function sync() {
  TRACKER.session_end = new Date().toISOString();
  await fetch('/api/attendance/mobile/activity-log', {
    method: 'POST',
    headers: { 'Authorization': 'Bearer '+token, 'Content-Type': 'application/json' },
    body: JSON.stringify({ attendance_date: today, ...TRACKER })
  });
}
setInterval(sync, 5 * 60000);
window.addEventListener('beforeunload', sync);
