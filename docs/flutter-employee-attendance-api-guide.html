Flutter Employee Attendance App API Guide

This document is the direct handoff for the employee mobile app. The app scope is only: login, profile/home page, check-in, check-out, active attendance status, and attendance history.

Login → Fetch profile/bootstrap → Show home screen → Check In / Check Out → Show active session → Show attendance history

1. App Scope

Login

Employee logs in with email or phone number and password.

Home

Show profile, branch, shift, current policy, today attendance state, and live session.

Attendance

Allow check-in and check-out with GPS, selfie, device ID, request IP, and remarks.

4. APIs To Use

POST/auth/login

Login using email or phone number.

{ "login": "employee@gmail.com", "password": "employee@123", "remember": true }

Success response:

{ "status": "success", "message": "Login successful", "access_token": "TOKEN_VALUE", "token_type": "Bearer", "expires_at": "2026-05-09T08:00:00+00:00", "user": { "id": 12, "uuid": "user-uuid", "name": "Rahul Das", "email": "employee@gmail.com", "phone_number": "9000000002", "role": "employee", "status": "active" } }

GET/auth/profile

Get logged-in user profile after login.

Header:

Authorization: Bearer TOKEN_VALUE

GET/attendance/mobile/bootstrap

Use this immediately after login to build the employee home screen.

Main things returned:

  • Company settings
  • Employee profile
  • Assigned branch
  • Shift info
  • Attendance policy flags
  • Allowed branch IP patterns
  • Today attendance
  • Server time

Important fields to use on home page:

data.employee.name data.employee.employee_code data.employee.branch_name data.employee.shift_name data.employee.attendance_policy_name data.employee.gps_required data.employee.selfie_required data.employee.device_binding_required data.employee.geofence_required data.employee.wifi_ip_restriction_required data.employee.branch_networks data.today_attendance

GET/attendance/mobile/request-ip

Get the server-detected request IP before check-in or check-out.

{ "status": "success", "data": { "request_ip": "49.36.120.10", "is_loopback": false, "hint": null, "server_time": "2026-05-08T12:20:11+05:30" } }

GET/attendance/mobile/active-session

Use on home page refresh and after a punch to show the active day session.

Main returned blocks:

  • data.attendance
  • data.tracks

POST/attendance/mobile/punch

Use for both check-in and check-out. This is the main attendance API.

Content type: multipart/form-data

Recommended payload:

punch_type = check_in | check_out attendance_mode = online work_mode = office | field | wfh | hybrid occurred_at = 2026-05-08T10:05:00+05:30 internet_status = online network_type = wifi remarks = Optional remark latitude = 22.5728 longitude = 88.3641 gps_accuracy_meters = 14 location_text = Kolkata Office Gate 1 device_id = flutter-device-001 device_name = Samsung Galaxy S24 device_platform = android device_model = SM-S921B app_version = 1.0.0 selfie = [multipart image file]

Sample success response:

{ "status": "success", "message": "Attendance punch processed successfully.", "data": { "attendance": { "id": 35, "attendance_date": "2026-05-08", "check_in_time": "2026-05-08 10:05:00", "check_out_time": null, "status": "present", "approval_status": "approved", "attendance_mode": "online", "work_mode": "office" }, "exceptions": [] } }

Possible exception values:

outside_geofence wifi_ip_not_allowed selfie_missing device_id_missing gps_missing

GET/attendance/mobile/history

Use for the attendance history page.

Supported query params:

?from=2026-05-01&to=2026-05-31&status=present&per_page=20

Response structure:

{ "status": "success", "data": [ { "id": 35, "attendance_date": "2026-05-08", "check_in_time": "2026-05-08 10:05:00", "check_out_time": "2026-05-08 18:08:00", "total_working_minutes": 483, "late_minutes": 5, "status": "late", "attendance_mode": "online", "approval_status": "approved" } ], "pagination": { "page": 1, "per_page": 20, "total": 24, "last_page": 2 } }

6. Recommended Flutter Check-In Flow

1. Load bootstrap 2. Load request-ip 3. Read current GPS 4. Read current device ID 5. Open direct camera capture 6. Collect selfie image 7. POST /attendance/mobile/punch with punch_type=check_in 8. Reload /attendance/mobile/active-session 9. Update home page state

7. Recommended Flutter Check-Out Flow

1. Load request-ip 2. Read current GPS 3. Reuse saved device ID 4. Open direct camera capture if policy requires checkout selfie 5. POST /attendance/mobile/punch with punch_type=check_out 6. Reload /attendance/mobile/active-session 7. Refresh history list if open

9. Minimal Flutter Modules Needed

Profile Module

Attendance Module

History Module

Fetch paginated attendance history with date/status filters.

10. Notes For Flutter Build

  • Use direct camera capture for selfie. Do not use file picker.
  • Use a persistent generated device ID and reuse it on every punch.
  • Before each punch, refresh the request IP from the backend.
  • Always send GPS if available.
  • Render SQL date/time fields in company timezone using AM/PM format.
  • The employee app scope does not need leave screens, notifications, or admin modules right now.
